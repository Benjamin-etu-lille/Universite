Slide 1 - Application Classification

Présentation du projet

Le projet a pour objectif de développer un outil permettant de charger, afficher et classifier des données en utilisant l’algorithme k-NN (k plus proches voisins). Ce projet offre une interface graphique permettant de visualiser des jeux de données sous forme de nuages de points.

Fonctionnalités principales

  • Chargement et affichage des données : L'utilisateur peut charger des fichiers de données et les visualiser sous forme de nuage de points. Le projet supporte des jeux de données comme Iris.
  • Projection des données : L'application permet à l'utilisateur de choisir les attributs à projeter sur les axes X et Y pour une exploration interactive des données.
  • Classification k-NN : L'outil permet de classifier des données non étiquetées en utilisant l'algorithme k-NN, avec la possibilité de choisir différentes distances pour la classification (par exemple, Manhattan ou Euclidienne).
  • Ajout de nouveaux points : Les utilisateurs peuvent ajouter de nouvelles données au nuage de points et choisir la catégorie de ces nouveaux points, qui seront ensuite intégrés à la visualisation.
  • Évaluation de la robustesse : L’application permet également d’évaluer la performance du modèle k-NN pour différents paramètres, notamment pour déterminer le meilleur choix de k.

Langages et outils : Java, JavaFX, Maven, Markdown

Slide 2 - Application Assistant Personnel de Voyage

Présentation du projet

Dans cette SAE, nous avons développé un logiciel en Java en utilisant le framework JavaFX. L'objectif principal était de calculer les k plus courts chemins entre deux villes saisies par l'utilisateur en prenant en compte différents modes de transport et critères comme les émissions de CO2 ou la durée. Les données des trajets étaient importées depuis un fichier CSV.

Le projet comprend un répertoire avec le logiciel prêt à être exécuté, une vidéo démontrant l'utilisation de l'application et des rendus graphiques analysant la problématique à travers différentes versions.

Slide 3 - Ventilation BDD Parcoursup & Statistiques

Présentation du projet

Cette SAE consistait en la création d'une base de données à partir des données fournies dans les archives de Parcoursup. Nous avons ventilé ces données afin de pouvoir les exploiter sans redondances. Le projet inclut :

  • Un script SQL de ventilation à exécuter avec PostgreSQL.
  • Un script SQL pour tester des requêtes de calcul.
  • Des rendus sur l'avancée du processus de ventilation et les statistiques obtenues.
  • Une analyse en anglais des données issues du Parcoursup anglais.

Parmi les conclusions obtenues, nous avons par exemple remarqué qu'il y a plus de garçons que de filles en informatique.

Langages et outils : SQL, PostgreSQL, Markdown
